# Noovel フォーマット仕様

Noovel が読み込めるファイルと、その扱いの一覧。

| 拡張子 | 扱い |
|---|---|
| `.noovel` | 正式フォーマット（JSON）。検証に通れば**推測ゼロ**でそのまま表示。検証に落ちたら警告を出して自動整形に降格（突き返さない） |
| `.txt` `.md` `.markdown` `.text` | 自動整形（ヒューリスティック）で章立て・表・見出しを推定して取り込み |
| `.webarchive` | Safariの「Webアーカイブ」保存ファイル。対応する小説ページなら本文・タイトル・著者・シリーズ・章を構造のまま取り込み。それ以外のページは本文テキストを抜いて自動整形 |
| 貼り付け | 本棚の「＋ 貼り付けで追加」から。貼った中身がJSONなら `.noovel` として検証取り込み（コードフェンス付きでも可）、それ以外は自動整形 |

## `.noovel`（正式フォーマット / JSON）

```json
{
  "noovel": 1,
  "title": "作品タイトル",
  "subtitle": "副題（任意）",
  "author": "著者名（任意）",
  "chapters": [
    {
      "title": "第一章　章タイトル",
      "blocks": [
        { "t": "p", "text": "段落。1ブロック=1段落。" },
        { "t": "h", "text": "章の中の小見出し" },
        { "t": "gap" },
        { "t": "table", "rows": [["見出しA", "見出しB"], ["値1", "値2"]] }
      ]
    }
  ]
}
```

### ブロックの種類

| t | 意味 | 必須フィールド |
|---|---|---|
| `p` | 段落 | `text` |
| `h` | 節見出し（章の中の小見出し。目次には出ない） | `text` |
| `gap` | 場面転換の空き | なし |
| `table` | 表。先頭行がヘッダー | `rows`（文字列の二次元配列） |

- 簡易形として `blocks` の代わりに `"text": "本文\n本文"`（改行区切り）も許容。1行が1段落になる。
- 検証ルール: `title` 必須・`chapters` 1つ以上。落ちたらエラーで止めず、警告つきで自動整形に切り替える。

## 自動整形（.txt / .md）のルール概要

- **章見出しの検出**: `第○章` / `## ` / `【】` / `■` / `1.` などのスタイルを文書全体で数え、**一貫して使われているスタイルだけ**を見出しとして採用（1行だけ紛れた記号では誤爆しない）
- 記号なしのファイルは「空行に続く短い行」から章を推定（推定したときは取り込み時にその旨を表示）
- 上位スタイル＝章（目次に出る）、下位スタイル＝節見出し（本文中の装飾）
- 巻頭の短い行はタイトル・副題として吸い上げ。`著者: ◯◯` 行は著者名になる
- タブ区切り・markdown記法の表は表として描画
- markdownの `**強調**` は除去、`---` は場面転換に変換
- 文字コードは UTF-8 / Shift_JIS を自動判定

## AI生成用プロンプト（コピペ用）

ChatGPT などに文章を書かせて Noovel で読みたいとき、依頼文の最後にこれを貼る：

```
出力は以下のJSON形式（.noovel形式）にしてください。コードブロックで囲んで出力してください。

{
  "noovel": 1,
  "title": "作品タイトル",
  "subtitle": "副題（なければ空文字）",
  "author": "著者名（なければ空文字）",
  "chapters": [
    {
      "title": "章タイトル",
      "blocks": [
        { "t": "p", "text": "段落テキスト。1ブロック=1段落にする" },
        { "t": "h", "text": "章内の小見出し（必要な場合のみ）" },
        { "t": "table", "rows": [["列1", "列2"], ["値", "値"]] }
      ]
    }
  ]
}

ルール:
- 章は "chapters" の要素として分ける（1章あたり2000〜5000字目安）
- 段落は必ず "p" ブロックに分割する（1つのpに複数段落を詰めない）
- 表で見せたい情報は "table" ブロックを使う
- JSON以外の説明文は出力しない
```

出力されたJSONを `○○.noovel` という名前で保存して Noovel で開くか、そのまま「＋ 貼り付けで追加」に貼る（JSONは自動で `.noovel` として検証取り込みされる）。
